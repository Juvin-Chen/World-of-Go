### 阶段一：语法基础与核心特性 (耗时：约 1-2 周)

**目标：** 能够熟练使用 Go 写出规范的代码，理解 Go 与 C++ 的不同。

1. **环境配置**
   - 在 VS Code 中安装 **Go 官方插件**。
   - 配置 Go Modules (`go mod`)，这是 Go 的依赖管理工具，必须掌握。
2. **基础语法 (对比 C++ 学习)**
   - **变量与切片 (Slice)**: Go 的数组很少用，主要用 Slice（动态数组）。注意 Slice 的扩容机制（面试常问）。
   - **指针**: Go 有指针，但不支持指针运算，比 C++ 安全。
   - **Map**: 字典的使用，注意 Map 不是并发安全的（并发读写会 panic）。
   - **Struct & Method**: Go 没有 `class`，只有 `struct`。
3. **核心难点 (面试重灾区)**
   - **Interface (接口)**: Go 的接口是**隐式实现**的（Duck Typing），这与 C++ 的虚函数继承完全不同。这是解耦的关键。
   - **Goroutine (协程)**: Go 的杀手锏。理解它比线程轻量在哪里（用户态调度）。
   - **Channel (通道)**: "不要通过共享内存来通信，要通过通信来共享内存"。学习 buffered vs unbuffered channel。
   - **Defer**: 延迟执行，常用于资源释放（类似 C++ 的析构，但更灵活）。
   - **Error Handling**: 习惯 `if err != nil`，这是 Go 的特色。

> **推荐资源**:
>
> - **A Tour of Go**: 官方自带的交互式教程，必刷。
> - **Go by Example**: 查代码片段的神器。

### 阶段二：Web 开发与标准库 (耗时：约 1-2 周)

**目标:** 不依赖框架，用标准库写一个简单的 HTTP Server，理解底层原理。

1. **标准库 `net/http`**
   - 如何启动一个 Server。
   - Request 和 Response 的结构。
   - **Context 包**: **(非常重要！)** 用于控制超时、取消请求和跨 API 传递数据。面试必问。
2. **数据处理**
   - `encoding/json`: JSON 的序列化与反序列化（Struct tags 的用法）。
3. **数据库连接**
   - **SQL**: 学习 `database/sql` 标准接口。
   - **ORM**: 学习 **GORM** (Go 圈最流行的 ORM 库)。即使你会 SQL，实习工作中大部分时间在用 ORM。

------

### 阶段三：主流框架与工程化 (耗时：约 2 周)

**目标:** 掌握企业级开发工具，能够开发像样的 API。

1. **Web 框架**
   - **Gin**: **(首选推荐)** 目前最流行的 Go Web 框架，轻量、性能好。实习面试大概率问这个。
   - *学习重点*: 路由组 (Route Group)、中间件 (Middleware) 的编写（鉴权、日志、跨域）。
2. **常用中间件/存储**
   - **Redis**: 必须掌握。用于缓存、Session 存储。学会 `go-redis` 库。
   - **JWT**: 用户认证标准，学会如何在 Gin 中实现 JWT 登录拦截。
3. **配置管理**
   - **Viper**: 用于读取配置文件（yaml/toml）。

------

### 阶段四：分布式与微服务基础 (加分项/进阶)

**目标:** 这是一个区分"普通学生"和"优秀实习生"的分水岭。

1. **RPC 通信**
   - 了解 **gRPC** 和 **Protobuf**。现代 Go 后端基本都是微服务，内部通信不用 HTTP，用 gRPC。
2. **Docker**
   - 学会编写 `Dockerfile`，把你的 Go 程序打包成镜像运行。
3. **并发模型实战**
   - 尝试写一个**Worker Pool**（工作池），理解如何控制并发数量，防止把 CPU 打满。

### 💡 推荐的项目方向 (简历必备)

为了找实习，你至少需要 **1-2 个包含源码的 GitHub 项目**。不要只写"图书管理系统"，太烂大街了。

1. **项目 A：高性能短链接生成系统 (侧重高并发/缓存)**
   - **核心功能**: 长链接转短链接，支持有效期，访问统计。
   - **技术栈**: Gin + GORM + **Redis (布隆过滤器)** + MySQL。
   - **亮点**: 解决"缓存击穿/穿透"问题，使用 MurmurHash 算法。
2. **项目 B：即时通讯 (IM) 聊天室 (侧重 WebSocket/协程)**
   - **核心功能**: 单聊、群聊、离线消息。
   - **技术栈**: Gin + **WebSocket** + Goroutine + Channel。
   - **亮点**: 每个连接一个 Goroutine，利用 Channel 分发消息，体验 Go 的并发优势。
