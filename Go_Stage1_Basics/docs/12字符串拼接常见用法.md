# Go 语言字符串拼接 

## 一、核心前提

Go 中`string`是**不可变类型**（空值为""，而非nil），每次直接拼接（如`+`）都会创建新字符串，因此不同场景要选不同拼接方式，平衡易用性和性能。

## 二、4 种常用拼接方式（按优先级 / 常用度排序）

### 1. strings.Builder（推荐 / 高性能）

#### 核心特点

- 官方推荐的高性能拼接方式，内部维护字节切片，避免频繁创建新字符串；
- 支持预分配容量（`Grow`），进一步提升性能。

#### 代码示例

```go
package main

import (
    "fmt"
    "strings"
)

func main() {
    // 1. 基础用法
    var builder strings.Builder
    builder.WriteString("Hello")   // 追加字符串
    builder.WriteString(" ")
    builder.WriteString("Go!")
    result := builder.String()     // 最终拼接结果
    fmt.Println(result) // 输出：Hello Go!

    // 2. 进阶：预分配容量（减少扩容，性能更优）
    var builder2 strings.Builder
    builder2.Grow(10) // 预估总长度，提前分配10字节内存
    builder2.WriteString("Hi, ")
    builder2.WriteString("World")
    fmt.Println(builder2.String()) // 输出：Hi, World
}
```

#### 适用场景

- 大量字符串拼接（如循环拼接、动态拼接）；
- 对性能有要求的场景（日常开发首选）。

### 2. + 运算符（简单 / 低性能）

#### 核心特点

- 语法最简单，直观易懂；
- 每次拼接都会创建新字符串，性能差（尤其循环中）。

#### 代码示例

```go
package main

import "fmt"

func main() {
    s1 := "Hello"
    s2 := " "
    s3 := "Go"
    result := s1 + s2 + s3
    fmt.Println(result) // 输出：Hello Go
}
```

#### 适用场景

- 少量字符串拼接（2-3 个）；
- 临时、简单的拼接场景（无需考虑性能）。

### 3. fmt.Sprintf（格式化 / 灵活）

#### 核心特点

- 支持格式化拼接，可混合字符串、数字、布尔值等多种类型；
- 性能低于`strings.Builder`，但灵活性高。

#### 代码示例

```go
package main

import "fmt"

func main() {
    name := "张三"
    age := 20
    score := 95.5
    // 按格式拼接不同类型数据
    result := fmt.Sprintf("姓名：%s，年龄：%d，分数：%.1f", name, age, score)
    fmt.Println(result) // 输出：姓名：张三，年龄：20，分数：95.5
}
```

#### 适用场景

- 需要格式化的拼接（如混数字、小数、布尔值）；
- 拼接逻辑简单且格式要求高的场景。

### 4. strings.Join（切片拼接）

#### 核心特点

- 专为字符串切片（[] string）设计，支持自定义分隔符；
- 拼接切片时比`+`更高效。

#### 代码示例

```go
package main

import (
    "fmt"
    "strings"
)

func main() {
    strSlice := []string{"apple", "banana", "orange"}
    // 带分隔符拼接
    result1 := strings.Join(strSlice, ", ")
    fmt.Println(result1) // 输出：apple, banana, orange
    
    // 无分隔符拼接
    result2 := strings.Join(strSlice, "")
    fmt.Println(result2) // 输出：applebananaorange
}
```

#### 适用场景

- 拼接字符串切片（[] string）；
- 需要统一添加分隔符的拼接场景（如逗号、空格分隔）。

## 三、各方式对比表

| 方式            | 优点                           | 缺点                     | 核心适用场景                |
| :-------------- | :----------------------------- | :----------------------- | :-------------------------- |
| strings.Builder | 性能最优，内存开销小，可预分配 | 代码量略多于`+`          | 大量 / 循环拼接（日常首选） |
| + 运算符        | 语法最简单，易理解             | 性能差，频繁创建新字符串 | 少量（2-3 个）字符串拼接    |
| fmt.Sprintf     | 支持格式化，可混合多种类型     | 性能较差                 | 格式复杂的拼接（混数字等）  |
| strings.Join    | 适配字符串切片，支持分隔符     | 仅适用于 [] string 类型  | 拼接字符串切片              |

## 四、避坑 & 最佳实践

1. **禁止**在循环中使用`+`拼接字符串（性能极差，会频繁创建临时字符串）；
2. 大量拼接优先用`strings.Builder`，并尽量用`Grow(n)`预分配容量；
3. 格式化拼接优先选`fmt.Sprintf`，而非先转字符串再用`+`拼接；
4. 拼接切片时，`strings.Join`比手动循环`+`更高效。

### 总结

1. 日常开发**首选`strings.Builder`**（高性能，适配 90% 以上场景）；
2. 少量拼接用`+`，格式化拼接用`fmt.Sprintf`，切片拼接用`strings.Join`；
3. 核心原则：根据「拼接数量」「是否格式化」「数据类型」选择对应方式，优先保证性能。
